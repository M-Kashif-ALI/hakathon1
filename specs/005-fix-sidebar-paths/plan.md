# Implementation Plan: Fix Sidebar Document Paths

**Feature**: 005-fix-sidebar-paths
**Created**: 2025-12-21
**Status**: Draft
**Author**: Claude

## Technical Context

This plan addresses the Docusaurus error where document IDs in the sidebar configuration don't match the actual document IDs generated by Docusaurus. The error shows that Docusaurus expects IDs without numeric prefixes (e.g., 'isaac-robot-brain/...') but the sidebar was configured with numeric prefixes (e.g., '003-isaac-robot-brain/...').

### Architecture Overview

The fix involves updating the sidebar configuration to use document IDs that match what Docusaurus generates. This is a configuration-only change that follows Docusaurus conventions for document ID generation.

### Technology Stack

- **Documentation Framework**: Docusaurus
- **Configuration File**: sidebars.ts
- **Document ID Convention**: Docusaurus standard (ignoring numeric prefixes in folder names)

### Dependencies

- Understanding of Docusaurus document ID generation patterns
- Existing Module 3 content in my-book/docs/003-isaac-robot-brain/
- Docusaurus documentation framework

### Integration Points

## Phase 1: Design & Architecture

### 1.1 Configuration Update Design

The fix requires updating the sidebars.ts file to use document IDs that match Docusaurus conventions:

```
Current (incorrect): '003-isaac-robot-brain/isaac-sim/index'
Correct (expected):  'isaac-robot-brain/isaac-sim/index'
```

### 1.2 Implementation Approach
- Update all Module 3 document references in sidebar to exclude numeric prefixes
- Maintain proper hierarchical structure for Isaac Sim, Isaac ROS, and Nav2 chapters
- Use consistent labeling and organization patterns
